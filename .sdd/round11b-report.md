## Concerns
1. **Seed-скрипт не оновлює наявні ключі**: `seedTranslations()` в backend вставляє лише ВІДСУТНІ пари (langCode, key), не апдейтить існуючі значення. Для `createGame.prizePoolLabel` (перейменування наявного ключа, п.6) я одноразово оновив рядки напряму в БД (docker-контейнер `tttm-mysql-1`) через тимчасовий ts-node скрипт (створений і одразу видалений, не залишився в репо, коміту не було) — інакше зміна в seed-файлі не проявилась би без ручного апдейту БД чи окремої seed:update-логіки. Раджу оркестратору мати на увазі: при деплої/реседі на інших середовищах (staging/prod) значення `createGame.prizePoolLabel` в БД теж потрібно буде оновити вручну (UPDATE), просто `npm run seed:i18n` це не підхопить.
2. Спільний Playwright-браузер: під час верифікації сесія MCP виявилась спільною з паралельним агентом (інші вкладки на порту 3121 з іншим станом гри) — я працював виключно у власній вкладці (порт 3122/create-game, явно вибраній/перевіреній перед кожною дією), тож на роботу іншого агента це не вплинуло, але варто знати про цю крихкість середовища.
3. Для п.9 нових i18n-ключів не додавав — наявна валідація (ім'я/ціна/час) вже покривала всі згадані в брифі кейси (порожнє ім'я, ціна поза межами, дата в минулому) через існуючі ключі `nameError`/`priceError`/`timeError`.
